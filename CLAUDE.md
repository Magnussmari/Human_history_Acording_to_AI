# Human History According to AI — Project Rules

> 5,226-year structured chronicle of human civilization. Layer 1 (year-level JSON corpus) shipped 2026-04-13. Layer 2 (scholarly evidence layer) in progress on `evidence-layer` branch.

## TOP-PRIORITY RULES (read first)

1. **Never modify `RESEARCH_PROMPT.md`.** It is the canonical ICCRA prompt and the contract behind all 5,226 JSON files. Any change invalidates the corpus. (See `CONTRIBUTING.md:43`.)
2. **Never touch `outputs/json/*.json` by hand.** These are daemon outputs. Fixes flow through `scripts/fix_categories.py`, `scripts/backfill_meta.py`, or a new re-research run — never sed/jq edits in place.
3. **Schema is enforced.** `scripts/validate_corpus.py` is the gate. After any corpus change, run it and confirm 100% valid before committing.
4. **Never swap models without an archived quality comparison.** Haiku and Gemini Flash experiments both failed quality checks; archived under `outputs/haiku_experiment/` and `outputs/gemini_experiment/`. Sonnet 4.6 is the chosen model; changing it requires a side-by-side run.
5. **Frontend must never crash on unexpected AI enum values.** Categories/certainties occasionally drift outside the schema. Any lookup against `CATEGORY_CONFIG` / `CERTAINTY_CONFIG` / docLevel maps must guard for unknowns (see commits `fix: guard ...`).
6. **Next.js 16 is NOT the Next.js in your training data.** Before writing frontend code, read `frontend/node_modules/next/dist/docs/` for the current API surface. Heed deprecation notices. (See `frontend/AGENTS.md`.)
7. **Priority order:** correctness > honesty-about-uncertainty > clarity > maintainability > performance. The project's whole value proposition is "every gap is declared, nothing is fabricated" — do not paper over uncertainty.
8. **Ask before committing.** Daemon-style auto-commits (`Progress: …`, `Auto-refresh: …`) are written by `scripts/git_sync.sh` and `scripts/refresh_frontend.sh`. Human commits should use the observed lowercase-prefix style (`docs:`, `fix:`, `feat:`, `redesign:`) with em-dash elaboration.

## CORPUS SCHEMA (ICCRA)

Every year JSON must pass `validate_corpus.py`. Required top-level keys: `year` (int), `year_label` (str with "CE"/"BCE"), `era_context`, `documentation_level` ∈ {rich,moderate,sparse,minimal,negligible}, `geographic_coverage_gaps` (array), `events` (array; may be empty), `disconfirming_evidence` (non-empty string), `historiographic_note`, `graph_edges`. Every event needs `id`, `title`, `region`, single-value `category` ∈ {political, military, scientific, cultural, economic, demographic, technological, religious, environmental, exploration, legal}, `description`, `key_figures`, `sources` (each with `name`, `type`, `contemporary`), single-value `certainty` ∈ {confirmed, probable, approximate, traditional, legendary}.

Known drift: compound categories like `"political | military"` occur occasionally and are normalized by `scripts/fix_categories.py`. Do not defend this upstream — keep the schema strict.

## ANTI-PATTERNS (evidence-backed)

- **Mocking/stubbing schema-critical fields.** The project's value is honesty. Empty `events: []` with real `era_context` beats fabricated entries.
- **Stripping `disconfirming_evidence`.** Anti-sycophancy protocol is mandatory; "none identified" is acceptable, empty string is not.
- **Using `claude -p` CLI for bulk research.** It adds ~3.5× token overhead (README Phase 3). Use `scripts/api_client.py` direct API calls.
- **Editing `outputs/json/` with jq/sed to "fix" issues.** Fix via a re-researchable migration script or delete + re-run the year; preserve the audit trail.
- **Committing to main from the `evidence-layer` branch.** Currently that branch has 76 staged files + commander-pending review.

## COMMANDS

### Python / daemon
- `python3 scripts/validate_corpus.py` — full ICCRA validation (expects 100% valid)
- `python3 scripts/fix_categories.py` — normalize compound categories
- `python3 scripts/backfill_meta.py` — backfill missing `_meta` blocks
- `bash scripts/health_check.sh` — daemon status snapshot

### Frontend
- `cd frontend && npm run dev` — Next.js dev server (port 3000)
- `cd frontend && npm run build` — production build
- `cd frontend && npm run lint` — ESLint
- `node frontend/scripts/aggregate-data.mjs` — regenerate 100-year chunks + manifest (run from repo root)

### Deployment
- Vercel auto-deploys `frontend/` on push to `main`
- Daemon runs on Hercules VM via `docker/docker-compose.yml` (Phase 1 done; container may still be running idle)

## HANDLING AI-GENERATED DATA IN UI

Every frontend lookup against an enum config must fall back safely:
```ts
const config = CATEGORY_CONFIG[event.category] ?? CATEGORY_CONFIG.political;
```
Observed historical crashes came from AI outputting `"political | military"` or `"documentation"` (instead of a docLevel). Guard every map access.
